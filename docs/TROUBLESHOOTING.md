# Решение проблем

## cfgutil не найден

Установите Apple Configurator и Automation Tools. После установки откройте **Настройки** приложения и нажмите **Проверить снова**.

## Mac не переходит в DFU

Проверьте по порядку:

1. Host Mac работает на Apple silicon.
2. Кабель поддерживает передачу данных, а не только зарядку.
3. Кабель подключён напрямую, без USB-хаба или док-станции.
4. На Target Mac используется правильный DFU-порт из таблицы Apple.
5. Target Mac подключён к питанию, если это настольная модель.
6. Xcode Command Line Tools установлены.

Если повторная попытка не помогла, используйте ручную последовательность, показанную в разделе **Режим DFU**.

## IPSW не появляется

- сначала дождитесь обнаружения модели и ECID;
- нажмите **Обновить**;
- проверьте доступ в интернет;
- временно выберите стандартный источник IPSW.me;
- для внутреннего источника проверьте формат и доступность JSON-каталога.

## Загрузка остановилась

Откройте раздел **Загрузки** и нажмите **Продолжить**. Приложение сохраняет данные для продолжения между запусками, если сервер поддерживает докачку.

## Restore завершился ошибкой

Убедитесь, что IPSW предназначен для точной модели и всё ещё подписывается Apple. Переподключите кабель, снова переведите Mac в DFU и повторите операцию. Для beta-версий возможность установки может меняться независимо от каталога.

## Где находятся журналы

```text
~/Library/Application Support/Target Mac DFU/launcher.log
~/Library/Logs/TargetMacDFU.log
```

При создании Issue приложите диагностический архив, но предварительно проверьте его содержимое на наличие данных, которыми не хотите делиться публично.
